ALBUM PLUGIN 2.0+ -> 2.X UPGRADE INSTRUCTIONS

  1. If you are upgrading from v1.4 or below, please use the instructions further down on this page. Make a COMPLETE BACKUP before you begin.
  2. Upload all of the files and folders contained in the "/files" directory in this zip to your server. Do not upload install.html, license.html, upgrade.html, or changelog.html.
    - Upload everything in ASCII mode.
    - Be sure to set full permissions (CHMOD 777) on all the files in the "/templates" directory.
    Note: If you have made modifications to your SocialEngine installation, refer to changelog.html for a list of files and language variables modified in each version. If you have made changes to those files, we suggest using WinMerge to consolidate your changes with the new files. Make sure that you also upload "/admin/install_album.php".

  3. Login to your SocialEngine admin panel. On the "View Plugins" page you will see a link that will allow you to upgrade. Click the upgrade link. Your upgrade is now complete!

  4. Be sure to delete the file "admin/install_album.php" when you are finished. Leaving this file intact poses a serious security risk!

ALBUM PLUGIN 1.X -> 2.X UPGRADE INSTRUCTIONS

  1. The upgrade to 2.00 (or beyond) from 1.4 (or below) involves replacing all of your PHP, TPL, and CSS files. This means that you will lose any changes you have made to these files. Unfortunately there is no easy way to avoid this since we've made so many changes to the platform. If possible, make a note about each change you've made in your current installation so that you can reapply them after the upgrade process.

  2. Please note that you will not lose your settings or user data. Once you have finished uploading the files to your server, log into your SocialEngine admin panel. On the "View Plugins" page, you will see a link that will allow you to upgrade to 2.X. This will import all your settings and user data. The only data that will be lost is your users' profile styles and their email notification settings - unfortunately those could not be easily imported into the new platform.

  3. Be sure to delete the file "admin/install_album.php" when you are finished. Leaving this file intact poses a serious security risk!
